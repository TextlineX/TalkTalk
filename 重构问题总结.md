## 一、项目架构

```
TalkTalk/
├── TalkTalk/                  ← 前端（Vue 3 + Ant Design Vue）
│   ├── src/view/              ← login.vue / register.vue / saver.vue
│   ├── src/components/content/ ← news / home / post / classify / my 等 10 个页面
│   └── TalkTalk-Backend/      ← 后端（Express + MySQL，部署 Vercel）
```

## 二、致命问题

### 1. 前端强依赖后端，无后端则功能全废

| 页面 | 请求路径 | 后端是否存在 | 结果 |
|------|---------|-------------|------|
| `login.vue` | `POST /login` | ✅ | 需真实后端 |
| `register.vue` | `POST /register` | ✅ | 需真实后端 |
| `post.vue` | `POST /db` | ✅ | 需真实后端 |
| `classify.vue` | `GET /getClassify` | ✅ | 需真实后端 |
| `news.vue` | `POST /getArticle` | ❌ 不存在 | 页面空数据 |
| `home.vue` | `POST /getArticle` | ❌ 不存在 | 页面空数据 |

### 2. 后端依赖 MySQL，环境配置复杂

后端 `database.js` 需要四个环境变量：

```
sql_host / sql_user / sql_password / sql_database / sql_port
```

- 本地需要自建 MySQL
- Vercel 部署需要手动配环境变量
- 配置错误直接 500，无法演示

## 三、安全问题

### 3. SQL 注入漏洞（database.js）

```js
// ❌ 直接拼接字符串
const sql = `select * from user where NAME='${username}'`

// ✅ 应该用 prepared statement
const sql = 'SELECT * FROM user WHERE NAME = ?'
const [rows] = await pool.execute(sql, [username])
```

涉及文件：`database.js:131 / 145 / 149 / 207 / 217 / 228`

### 4. 外键约束错误（database.js）

```js
const adminTable = `CREATE TABLE IF NOT EXISTS admin (
    id INT(20) NOT NULL PRIMARY KEY AUTO_INCREMENT,
    level INT(2) NOT NULL,
    CONSTRAINT fv_admin FOREIGN KEY (level) REFERENCES user(id)  -- ❌ 外键引用 user(id)，但 user.id 是 INT(20)，level 是 INT(2)，类型不匹配
)`;
```

### 5. 注册密码明文存储（database.js）

数据库直接存明文密码，没有任何哈希处理。

## 五、代码质量问题

### 11. 双重嵌套 Promise（home.vue）

```js
fetch(...).then(res => {
  async function get_Response(){
    let vf = await res.json();  // ⚠️ 没必要再包 async function
    loadA(vf)
  }
  get_Response();
})
```

### 12. 直接操作 DOM

```js
// news.vue：页面初始化用 createElement + innerHTML 注入
let wrapper = document.querySelector('.wrapper');
let box = document.createElement('div');
box.innerHTML = `...`
wrapper.insertBefore(box, wrapper.children[0])

// home.vue：同样问题
```

Vue 的响应式没用上，完全用 DOM 操作写视图，违背 Vue 设计理念。

### 13. `init` 逻辑多余（main.js）

```js
let init = () => {
    if(localStorage.getItem('router') == null) {
        localStorage.setItem('router', '1');
    }
}
```

这个 `router` key 之后没有任何地方读取，完全无用代码。

## 八、重构方向建议

**推荐方案：前端内置 Mock 数据层**

- 删除 Element Plus，统一 Ant Design Vue
- 写 `src/api/mock.js` 内置示例数据，替代后端
- 各页面接入 mock，移除 `VITE_BACKEND_URL` 依赖
- 项目可直接部署 GitHub Pages，HR 点开即看
- 后续切回真实后端只需改 baseURL

**可选方案：做成完整全栈**

- 保留后端，修复安全漏洞
- 迁移到 SQLite（去除 MySQL 依赖）
- 部署简化
- 工作量大，能体现全栈能力
